1. What did you learn about using Git and GitHub?: 
    I learned "a lot" (relative to my previous knowledge) about using Git and GitHub. As someone who has used GitHub a lot as a consumer, not creator, it's cool to learn about what processes go into the projects I consume. 

2. What was most confusing or frustrating?:
    I didn't think anything was too confusing or frustrating. I think it was pretty straight forward, however this may be because I have some limited knowledge of linux terminal commands, which is what Git seems to be using in one way or another. 

3. Why do you think developers use Git?:
    There are probably a good amount of reasons why developers use Git, but the most apparent to me is that it is essentially a detailed archive of development, as well as being the de facto platform for
    uploading development projects.

4. How might this help with larger projects in the future?:
    I think it all scales. We need to take the smaller steps (like building a small JS file with trivial functions, and adding README files) in order to be able to take the larger steps.
